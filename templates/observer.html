Uso y Justificación de Observer

Patrón Observer: Gestión de notificaciones en tiempo real

El patrón de diseño Observer es una solución ampliamente utilizada en sistemas donde es necesario mantener la sincronización entre múltiples objetos, permitiendo que los cambios en un objeto se reflejen automáticamente en otros relacionados. Este patrón es ideal para implementar sistemas de notificaciones en tiempo real, asegurando que los suscriptores reciban actualizaciones de manera eficiente y estructurada.

En este caso, el patrón Observer se utiliza para gestionar notificaciones relacionadas con actividades académicas. Por ejemplo, un alumno puede ser notificado cuando se le asigna una nueva tarea, o un profesor puede recibir una notificación cuando un alumno entrega una actividad. Este enfoque garantiza que todos los usuarios relevantes estén informados de los eventos importantes en el sistema.

El patrón se implementa mediante las siguientes clases principales:

  • Publicador: Responsable de gestionar la lista de suscriptores y notificarles cuando ocurre un evento.
  • Suscriptor: Clase base abstracta que define la interfaz para los suscriptores.
  • SuscriptorAlumno: Implementación concreta de un suscriptor que representa a un alumno.
  • SuscriptorProfesor: Implementación concreta de un suscriptor que representa a un profesor.

El uso del patrón Observer en este contexto mejora la modularidad y escalabilidad del sistema, permitiendo agregar nuevos tipos de suscriptores o eventos sin afectar el código existente. Además, fomenta un diseño desacoplado, ya que los publicadores no necesitan conocer los detalles específicos de los suscriptores, lo que facilita el mantenimiento y la evolución del sistema.

class Publicador:
    """
    Clase que representa un publicador de eventos, que puede tener múltiples suscriptores.
    Esta clase permite agregar y eliminar suscriptores, así como publicar eventos a los suscriptores.

    Atributos:
    ---------
        suscriptores (list): Lista de suscriptores que recibirán los eventos.
    """

    def __init__(self):
        self.suscriptores = []

    def suscribir(self, suscriptor) -> None:
        """
        Agrega un suscriptor a la lista de suscriptores.
        Args:
            suscriptor (Suscriptor): El suscriptor a agregar.
        """
        self.suscriptores.append(suscriptor)
        print(f'Suscriptor {suscriptor} agregado.')

    def desuscribir(self, suscriptor) -> None:
        """
        Elimina un suscriptor de la lista de suscriptores.
        Args:
            suscriptor (Suscriptor): El suscriptor a eliminar.
        """
        self.suscriptores.remove(suscriptor)
        print(f'Suscriptor {suscriptor} eliminado.')

    def notificarSuscriptores(self, eventos : list = []):
        """
        Publica los eventos a todos los suscriptores.
        Recorre la lista de suscriptores y llama al método notificar de cada uno, pasando los eventos.
        Args:
            eventos (list): Lista de eventos a publicar.
        """
        for suscriptor in self.suscriptores:
            suscriptor.notificar(eventos)
