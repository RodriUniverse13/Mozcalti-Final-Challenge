Uso y Justificación de Strategy

Patrón Strategy: Implementación para la gestión de actividades

El patrón de diseño Strategy es una solución ampliamente utilizada en el desarrollo de software para permitir la selección dinámica de comportamientos en tiempo de ejecución. Este patrón resulta especialmente útil cuando se requiere implementar múltiples variantes de un algoritmo o funcionalidad, manteniendo un diseño flexible y extensible.

En este caso, el patrón Strategy se utiliza para gestionar diferentes tipos de actividades académicas, como exámenes, tareas y proyectos. Cada tipo de actividad tiene características y comportamientos específicos que se encapsulan en estrategias concretas, permitiendo que el sistema pueda adaptarse fácilmente a nuevos requisitos o tipos de actividades en el futuro.

Además, este patrón complementa el uso del patrón Builder, que se encarga de construir las asignaciones con sus respectivos atributos, como archivos adjuntos, formularios o preguntas. Una vez creada la asignación, el patrón Strategy permite interpretar y presentar la información de manera adecuada según el tipo de actividad, asegurando una experiencia de usuario coherente y eficiente.

En resumen, el uso del patrón Strategy en este contexto no solo mejora la organización y mantenibilidad del código, sino que también facilita la escalabilidad del sistema al permitir la incorporación de nuevos tipos de actividades sin afectar el funcionamiento existente.

from abc import ABC, abstractmethod
                            
class Estrategia(ABC):
    def __init__(self, asignacion):
        self.asignacion = asignacion

    @abstractmethod
    def mostrar(self):
        pass

class EstrategiaVerTarea(Estrategia):
    def __init__(self, asignacion):
        super().__init__(asignacion)

    def mostrar(self):
        return f"El nombre de la tarea es: {self.asignacion.nombre} y su descripcion es: {self.asignacion.descripcion}"
    
    def ver_archivos(self):
        return f"Los archivos de la tarea son: {self.asignacion.archivos}"

class EstrategiaVerExamen(Estrategia):
    def __init__(self, asignacion):
        super().__init__(asignacion)
        self.asignacion.questions = self.asignacion.questions or []

    def mostrar(self):
        return f"El examen es: {self.asignacion.nombre}, descripcion: {self.asignacion.descripcion}, fecha: {self.asignacion.fecha_entrega}"
    
    def mostrar_preguntas(self):
        return f"Las preguntas son: {self.asignacion.questions}"

class EstrategiaVerProyecto(Estrategia):
    def __init__(self, asignacion):
        super().__init__(asignacion)

    def mostrar(self):
        return f"El proyecto es: {self.asignacion.nombre}, descripcion: {self.asignacion.descripcion}, fecha: {self.asignacion.fecha_entrega}"
        
    def mostrar_archivos(self):
        return f"Los archivos son: {self.asignacion.archivos}"

class Manager:
    def __init__(self, estrategia):
        self.estrategia = estrategia
        
    def set_estrategia(self, estrategia):
        self.estrategia = estrategia
        
    def mostrar(self):
        return self.estrategia.mostrar()
